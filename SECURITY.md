# Security Policy

## Supported Versions

We currently support the latest deployment of the website. Previous versions are not maintained.

| Version | Supported          |
| ------- | ------------------ |
| Latest  | :white_check_mark: |
| Old     | :x:                |

## Reporting a Vulnerability

We take the security of `oralaundry.co.za` seriously. If you discover a security vulnerability, please follow these guidelines.

### How to Report

1.  **Do NOT open a public issue** on GitHub for sensitive security bugs.
2.  Please report the vulnerability by emailing us directly at  In your report, please include:
    * A description of the vulnerability.
    * Steps to reproduce the issue.
    * Screenshots or proof of concept (if safe to share).

### Response Process

* We will acknowledge your report within 48 hours.
* We will investigate the issue and determine the impact.
* We ask that you keep the vulnerability confidential until we have resolved the issue.

## Security Best Practices for Users

* Always access the site via **https://** (ensure the lock icon is visible).
* Report any suspicious behavior or phishing attempts related to our brand immediately.
